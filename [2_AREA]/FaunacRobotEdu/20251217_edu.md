# IO

Controller를 통해 Robot을 동작시키기 위한 전기 신호 통신.

- 범용 IO와  전용 IO 로 구분.
	- __전용 IO__: ROBOT I/O, SOP I/O, UOP I/O
	- __범용 IO__: Digital I/O, Group I/O, Analog I/O
- UOP I/O와  범용 I/O 는 팬던트에서 따로 Rack No, Slot No, Start No와  STAT(status)를 설정 필수.
- Rack No는 종류별로 사전 정의되어 있다.
	- CC-Link: 92
	- DeviceNet: 81~84
	- Ethernet: 89
- Slot No는 I/O 보드의  Slot 번호.
- Start No.
	- I/O 보드의 'I'의 경우 1~18까지는 UOP에 '할당'하는게 일반적이다.
	- I/O 보드의 'O'의 경우 1~20까지는 UOP에 '할당'하는게 일반적이다.
	- 그렇기 때문에 실제 DIO의 할당의 경우, Start No를  'I'의 경우  19 부터 할당, 'O'의 경우 21 부터 할당 하는게 일반적이다.

## 1. ROBOT I/O

 Controller와 Robot의 신호 정의. Tool의 클램프를 정의 한다.?

## 2. SOP I/O

표준조작반 (3 switch mode, emergency stop, ..) 신호를 정의 한다.

## 3. UOP I/O
	
PLC 신호를 정의 한다.
	
## 4. Digital I/O
	
사용자 신호를 정의를 한다.
	
## 5. Group I/O
	
여러 연속된 Digital I/O를 10진수로 정의한다. 연속된 순서대로 2진수의 값을 더한 값으로 정의한다.
	
## 6. Analog I/O
	
Analog 신호를 주고 받는 센서들을 정의한다.


# Macro

- 프로그램의 단축키 같은 역할을 함.  
- alias CALL 'PROGRAM' = 'MACRO' 구문을 추가한다고 이해하면 편함. 때문에 프로그램 내에서 'MACRO' 를 사용하여 CALL 효과를 볼 수 있음. 
- 팬던트에 SHIFT + 'KEY' 로 설정하고 바로 실행도 가능.
